# ADR-007: llama.h C API Shim Strategy

**Status**: Accepted
**Date**: 2026-03-01
**Deciders**: AutomatosX Team (DEFAI Private Limited)

---

## Context

ax-engine's `ax-shim` crate provides a `libllama.dylib` that matches llama.cpp's
`llama.h` C API. This enables C/C++ applications (python-llama-cpp, LM Studio,
Ollama's internal paths, custom bindings) to use ax-engine as a drop-in replacement
without recompilation.

The shim is well-designed:
- All unsafe pointer operations confined to the shim crate
- Opaque handles (`LlamaModel`, `LlamaContext`) — no layout exposure
- Null safety: every extern "C" function checks for null inputs
- Error semantics match llama.h conventions (0=success, -1=error)
- Header generated by `cbindgen`; never hand-edited

This design should be preserved in ax-serving.

---

## Decision

Port `ax-shim` to `ax-serving-shim` with the backend wired to `ax-serving-engine`'s
`InferenceBackend` trait instead of ax-core's `LlamaModel`.

### Handle Design

```rust
// ax-serving-shim/src/types.rs
pub struct LlamaModel {
    handle: ModelHandle,       // opaque ID from InferenceBackend
    backend: Arc<dyn InferenceBackend>,
    tokenizer: Arc<Tokenizer>, // from mistralrs tokenizers crate
    vocab_size: i32,
}

pub struct LlamaContext {
    model: Arc<LlamaModel>,
    position: usize,
    logits: Vec<f32>,
    rng_seed: u64,
    n_ctx: i32,
}
```

### Safety Invariants

1. Every `extern "C"` function begins with a null pointer check; returns -1 or null on null input
2. `Box::into_raw` / `Box::from_raw` are the only raw pointer operations
3. No raw pointer is stored across function boundaries (no aliasing)
4. `Arc<dyn InferenceBackend>` reference-counted; handles stay valid until `llama_free_model`
5. No `unwrap()` in the shim layer; all Results converted to error codes
6. Thread safety: `LlamaModel` is `Send + Sync`; `LlamaContext` is not (matches llama.cpp)

### Header Generation

```toml
# ax-serving-shim/cbindgen.toml
language = "C"
include_guard = "LLAMA_H"
tab_width = 4
documentation = true
```

CI gate: `cbindgen --verify` must pass (ensures header matches Rust signatures).

## Compatibility Test Suite

Port `tests/compat/` from ax-engine:

```
tests/compat/
  test_compat.c    — loads model, runs tokenize + eval + sample
  run_tests.sh     — drives test binary against golden outputs
```

The compat test must pass for every PR that touches `ax-serving-shim` or
`ax-serving-engine`.

## n_gpu_layers Mapping

`llama_model_params.n_gpu_layers` controls backend selection:

| `n_gpu_layers` | Backend |
|---|---|
| 0 | `AXS_BACKEND=cpu` |
| 1–999 | `AXS_BACKEND=metal` (full GPU, value ignored beyond "any GPU") |
| -1 | `AXS_BACKEND=metal` (all layers) |

## Consequences

### Positive

- **Drop-in compatibility**: existing C/C++ callers work without recompilation
- **Safety boundary**: all unsafe confined to one crate
- **Small shim**: ~800 lines; easy to audit
- **cbindgen**: header always in sync; never diverges from implementation

### Negative

- **Tokenizer coupling**: ax-serving-shim needs direct tokenizer access for
  `llama_tokenize` / `llama_token_to_piece`. mistralrs-core's `tokenizers` crate
  must be exposed through `ax-serving-engine`.
- **Sampling in shim**: llama.h's sampling API is stateful (mutates candidates array).
  The shim must implement this sampling logic locally rather than delegating to
  mistralrs-core's sampler (which is async-pipeline-based).
